 35  # A few notes on XNU's (64bit) memory layout: 
 36  # 
 37  # Because of the way the Darwin kernel (XNU) is bootstrapped, a section of its 
 38  # virtual address space maps linearly to the base of the physical address space. 
 39  # This relationship is basically: 
 40  # KERNEL_MIN_ADDRESS + the_physical_address = the_virtual_address 
 41  # 
 42  # The kernel ensures this when allocating certain data structures, most notably 
 43  # the page tables [1]. However, the kernel doesn't actually "know" the value of 
 44  # KERNEL_MIN_ADDRESS, which is defined the Makefile [2]. Instead, allocations 
 45  # are done by keeping a cursor at the lowest available physical address [3]. 
 46  # 
 47  # Because of this, when the kernel needs to convert an address from the virtual 
 48  # address space to the physical address space without relying on the page 
 49  # tables, it uses a "safer" variation on the above rule and masks out the first 
 50  # 32 bits of the address using a macro called ID_MAP_VTOP [4,5], which is a 
 51  # simple bitmask (LOW_4GB_MASK [6]). 
 52  # 
 53  # We copy/adapt all three #defines below. When we need to bootstrap the virtual 
 54  # address space, relying on ID_MAP_VTOP is preferrable, because it's less 
 55  # fragile. However, KERNEL_MIN_ADDRESS can be a good heuristic for deciding 
 56  # whether a particular value is a valid pointer in the kernel virtual address 
 57  # space, so I decided to keep it around. 
 58  # 
 59  # [1] 
 60  # github.com/opensource-apple/xnu/blob/10.9/osfmk/i386/i386_init.c#L134 
 61  # 
 62  # [2] 
 63  # github.com/opensource-apple/xnu/blob/10.9/makedefs/MakeInc.def#L258 
 64  # 
 65  # [3] This is where physfree is defined as the next free page, after a blank 
 66  # page, after the last page of the kernel image as determined by the bootloader. 
 67  # github.com/opensource-apple/xnu/blob/10.9/osfmk/i386/i386_init.c#L330 
 68  # 
 69  # [4] 
 70  # github.com/opensource-apple/xnu/blob/10.9/osfmk/i386/pmap.h#L353 
 71  # 
 72  # [5] Example use, to set the physical address of the DTB when switching address 
 73  # spaces, knowing the virtual address of the first page table: 
 74  # github.com/opensource-apple/xnu/blob/10.9/osfmk/i386/pal_routines.c#L254 
 75  # 
 76  # [6] 
 77  # github.com/opensource-apple/xnu/blob/10.9/osfmk/i386/pmap.h#L119
