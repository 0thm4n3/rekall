 40      Each entity consists of a an identity object, like PID, username or a memory 
 41      address, and a list of components, which are data about the entity. 
 42   
 43      Immutability: 
 44      ============= 
 45   
 46      Entities and components are copy-on-write. Components enforce immutability 
 47      while Entity and Identity are less strict for practical reasons. 
 48      Nevertheless, changing an instance of entity or identity after they've been 
 49      added to an entity manager will result in undefined behavior. 
 50   
 51      Relational Model: 
 52      ================= 
 53   
 54      Entites support associations with other entities. For example, a resource 
 55      has a handle, a handle is owned by a process, and so on. These associations 
 56      are implemented using Identity and lookups through an entity manager. See 
 57      documentation of 'find_referenced_entities' and 'find_referencing_entities' 
 58      for a discussion of associations. 
 59   
 60      Merging: 
 61      ======== 
 62   
 63      If more than one entity is added to an entity manager for the same thing, 
 64      they will be merged into a single entity with all the properties, as long as 
 65      the identity is decidable. For example, if we have a user with UID 15 and 
 66      another user with username "Alice" they will remain separate entities until 
 67      an entity is added with an identity that has the UID 15 and username 
 68      "Alice", at which point the identity becomes decidable and all three 
 69      entities will be merged into one. 
 70   
 71      If, during merging, a conflict arises then both versions are kept 
 72      encapsulated in an instance of superposition.Superposition. As long as you 
 73      use 'get_' family of accessors on Entity this will be handled for you (as 
 74      those functions always return generators). 
 75   
 76      Superpositions: 
 77      =============== 
 78   
 79      Superpositions are technically merge conflicts, but they are not errors. 
 80      They occur in cases where there are legitimately multiple valid values of a 
 81      single attribute. Some examples: 
 82   
 83      Resource/handle can have multiple values, because a single file/socket can 
 84      be opened by multiple handles owned by multiple processes. 
 85   
 86      Struct/type can be a superposition in case of unions, or things that 
 87      are stored as a void pointer and cast depending on contextual state. 
 88   
 89      User/real_name can often be a superposition because of variable formatting 
 90      rules applied by the OS. 
 91   
 92      Public state members: 
 93      ===================== 
 94   
 95      components: An instance of ComponentContainer, which is a fast container 
 96          object that actually stores the components of the entity. 
 97

 99
