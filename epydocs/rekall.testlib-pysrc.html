 23  """Base classes for all tests. 
 24   
 25  Rekall Memory Forensics tests have these goals: 
 26   
 27  - Detect regression bugs with previous versions of Rekall Memory Forensics. 
 28   
 29  - Once differences are detected, make it easier to understand why the 
 30    differences arise. 
 31   
 32  - Sometime the differences are acceptable, in that case, there need to be a way 
 33    to declare the allowed differences in the tests. 
 34   
 35  - What makes this more complicated is that the same test needs to be applied to 
 36    multiple images in a consistent way without necessarily making any code 
 37    changes to the test itself. 
 38   
 39   
 40  Solution Outline

 43  A baseline of running each test is written by the test suite itself. The user 
 44  can write a baseline for all the modules by issuing the make_suite binary. The 
 45  baseline for each module will generate a number of files in a given directory.
