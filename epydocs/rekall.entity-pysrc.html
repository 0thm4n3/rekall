 32      """Entity is an abstraction of things like Process, User or Connection. 
 33   
 34      Each entity consists of a an identity object, like PID, username or a memory 
 35      address, and a list of components, which are data about the entity. 
 36   
 37      Immutability: 
 38      ============= 
 39   
 40      Entities and components are copy-on-write. Components enforce immutability 
 41      while Entity and Identity are less strict for practical reasons. 
 42      Nevertheless, changing an instance of entity or identity after they've been 
 43      added to an entity manager will result in undefined behavior. 
 44   
 45      Relational Model: 
 46      ================= 
 47   
 48      Entites support associations with other entities. For example, a resource 
 49      has a handle, a handle is owned by a process, and so on. These associations 
 50      are implemented using Identity and lookups through an entity manager. See 
 51      documentation of 'find_referenced_entities' and 'find_referencing_entities' 
 52      for a discussion of associations. 
 53   
 54      Merging: 
 55      ======== 
 56   
 57      If more than one entity is added to an entity manager for the same thing, 
 58      they will be merged into a single entity with all the properties, as long as 
 59      the identity is decidable. For example, if we have a user with UID 15 and 
 60      another user with username "Alice" they will remain separate entities until 
 61      an entity is added with an identity that has the UID 15 and username 
 62      "Alice", at which point the identity becomes decidable and all three 
 63      entities will be merged into one. 
 64   
 65      If, during merging, a conflict arises then both versions are kept 
 66      encapsulated in an instance of superposition.Superposition. As long as you 
 67      use 'get_' family of accessors on Entity this will be handled for you (as 
 68      those functions always return generators). 
 69   
 70      Superpositions: 
 71      =============== 
 72   
 73      Superpositions are technically merge conflicts, but they are not errors. 
 74      They occur in cases where there are legitimately multiple valid values of a 
 75      single attribute. Some examples: 
 76   
 77      Resource.handle can have multiple values, because a single file/socket can 
 78      be opened by multiple handles owned by multiple processes. 
 79   
 80      MemoryObject.type can be a superposition in case of unions, or things that 
 81      are stored as a void pointer and cast depending on contextual state. 
 82   
 83      User.real_name can often be a superposition because of variable formatting 
 84      rules applied by the OS. 
 85   
 86      Public state members: 
 87      ===================== 
 88   
 89      identity: An instance of Identity, or subclass. 
 90   
 91      components: An instance of components.ComponentTuple - see that class for 
 92          details. 
 93   
 94      copies_count: The number of times this entity was discovered by different 
 95          collectors. Incremented with each merge. 
 96   
 97      collectors: A set of collector names that discovered this entity. 
 98   
 99      entity_manager: The manager this entity belongs to. 
100
